Sorting Algorithms

Bubble Sort

Compares adjacent elements and swaps them if they are in the wrong order, iterating until the entire list is sorted
Time complexity: O(n^2)

Insertion Sort

Builds the final sorted array one element at a time, comparing each element to its previous elements and shifting them to the right if they are greater.
Time complexity: O(n^2)

Merge Sort

A divide-and-conquer algorithm that continuously divides the input in half until each subarray has only one element, then merges them back together.
Time complexity:O(n log n)

quick Sort

Quick sort is the most optimized sort algorithms which performs sorting in O(n log n) comparisons. Like Merge sort, quick sort also work by using divide and conquer approach.
Time complexity: O(n log n)

selection Sort

Divides the input into a sorted and an unsorted region, repeatedly selects the smallest element from the unsorted region and swaps it with the leftmost element of the unsorted region.
Time complexity: O(n^2)

heap Sort

Heap Sort is a comparison-based sorting algorithm that uses a binary heap to repeatedly extract the maximum element and place it in the sorted portion of the array.
Time complexity: O(n log n)
